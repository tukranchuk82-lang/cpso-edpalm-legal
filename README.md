# ЦПСО EdPalm — Юридическая поддержка «Базовый без ГИА»

Прототип приложения-справочника: отвечает на частые юридические вопросы по
ОГЭ/ЕГЭ на семейном обучении и передаёт обращение живому юристу.

> **Это прототип фронтенда.** Реальные интеграции (DMS, SMS, Omnidesk, SaleBot)
> имитируются в браузере и помечены в коде как `[MOCK]`. Дизайн, экраны и вся
> пользовательская логика — настоящие.

---

## Как запустить

Сборка не нужна — это статический сайт.

1. **Через локальный сервер** (нужно для установки на рабочий стол и service worker):
   ```powershell
   # из папки проекта
   node server.js          # затем откройте http://localhost:5500
   # или: python -m http.server 5500
   ```
   Либо в VS Code — расширение **Live Server** → «Open with Live Server».

2. Можно просто открыть `index.html` двойным кликом, но тогда **не будет работать
   установка на рабочий стол** (PWA требует http/https, а не file://).

## 📲 Установка на рабочий стол (PWA)

Это сайт, который **выглядит и ставится как приложение** — без скачивания из
магазина. На рабочий стол добавляется ярлык-иконка (эмблема-герб).

- **Компьютер (Chrome/Edge):** кнопка «⬇️ Установить приложение» в самом приложении,
  либо значок установки в адресной строке.
- **Android (Chrome):** та же кнопка «Установить» → «Добавить на главный экран».
- **iPhone/iPad (Safari):** кнопка «Установить» покажет инструкцию: «Поделиться»
  ⬆️ → «На экран “Домой”».

> ⚠️ **Для боевого размещения нужен HTTPS** (на проде установка работает только по
> `https://`). На `localhost` всё работает для проверки.

Файлы PWA: `manifest.json` (имя, иконка, цвета), `sw.js` (service worker, офлайн),
иконка `assets/app-icon.svg`.

## Как проверить (демо-сценарии)

**Вход:** введите любой телефон, затем на экране кода доступ определяет код:

| Сценарий | Что сделать |
|---|---|
| ✅ Успешный вход | Любой телефон → код **`111111`** → доступ открывается |
| ❌ Нет доступа | Любой телефон → код **`000000`** → экран «Доступ не найден» |
| ↩️ Неверный код | Любой другой код → ошибка, поля очищаются |
| 📚 Темы и ответы | На главной выбрать тему → вопрос → готовый ответ |
| 💬 Вопрос юристу | «Задать вопрос юристу» → написать → «Отправить» |
| 🕒 Вне рабочего времени | Открыть форму в выходной / не 10–18 МСК — увидите текст-заглушку |

> Демо-коды задаются в `js/data.js` → `APP_CONFIG.demoCodes`.

## Что менять при доработке

### 1. Тексты ответов юриста — `js/data.js`
Ответы сейчас **черновые** (на главном экране ответа помечены плашкой
`⚠ черновик`). Замените поле `a` у каждого вопроса официальным текстом
из вашего документа / с сайта https://hss.center/. После замены уберите
`draft: true` — плашка исчезнет.

### 2. Персонажи — `assets/`
Сейчас Эврика и собака нарисованы как SVG. Чтобы поставить настоящие рендеры:
положите PNG в `assets/` и поменяйте `src` в `js/app.js` (константа
`characters`) и в подписи ответа.

### 3. Подключение реального backend — `js/app.js`
Найдите функции с пометкой `[MOCK]` и замените на вызовы вашего API:

| Функция-заглушка | Что подключить |
|---|---|
| `mockSendSms(phone)` | Отправка **SMS-кода** на номер |
| `mockVerifyCode(phone, code)` | Подтверждение кода + проверка доступа через **DMS** (пакет «Базовый без ГИА») |
| `mockSendToOmnidesk(payload)` | **Omnidesk** — передача обращения юристу |
| `resetTo('login')` в конце файла | Точка входа из **SaleBot** (Telegram / MAX / VK) |

### 4. График работы юриста — `js/data.js` → `APP_CONFIG.lawyerSchedule`
Дни, часы и часовой пояс (по умолчанию Пн–Пт 10:00–18:00 МСК).
Текст ответа вне рабочего времени — `APP_CONFIG.offHoursMessage`.
